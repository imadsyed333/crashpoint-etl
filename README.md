# Crashlog Geocode Pipeline

This project provides a geocoding pipeline for processing and analyzing collision and address data using Apache Airflow and Python. It is designed to automate the extraction, transformation, and geocoding of raw address and collision datasets, producing processed outputs for further analysis.

Rough work for developing this pipeline can be found [here](https://github.com/imadsyed333/geocrash).

## Project Structure

- `dags/` — Contains Airflow DAGs (workflows), e.g., `geocode_pipeline.py`.
- `src/` — Core pipeline logic and supporting Python scripts.
- `scripts/` — Initialization and helper scripts (e.g., `init_airflow.sh`).
- `data/` — Data directory (see below for details).
- `docker-compose.yaml` — Defines the Airflow service and dependencies.
- `requirements.txt` — Python dependencies for the pipeline.

## Data Directory

- `data/raw/` — Raw input files (e.g., `addresses.csv`, `collisions.csv`, `intersections.csv`).
  Note: you will need to obtain these on your own through the [City of Toronto Open Data Portal](https://open.toronto.ca/).

- `data/processed/` — Output files generated by the pipeline (e.g., `.parquet` files).

## Running the Project

1. **Install Docker and Docker Compose** (if not already installed).
2. **Start the Airflow environment:**
   ```sh
   docker compose up --build
   ```
   This will start Apache Airflow and mount the necessary folders.
3. **Access Airflow UI:**
   Open [http://localhost:8080](http://localhost:8080) in your browser.
4. **Trigger the pipeline:**
   Use the Airflow UI to trigger the `geocode_pipeline` DAG.
